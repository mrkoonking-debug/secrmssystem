# 📖 คู่มือการดูแลและแก้ไขระบบ (ฉบับเจ้าของทำเองได้)

คู่มือนี้เขียนขึ้นเพื่อให้เจ้าของระบบที่ **ไม่ได้เป็นโปรแกรมเมอร์** สามารถแก้ไขจุดเล็กๆ น้อยๆ ในระบบได้ด้วยตัวเองครับ โดยเน้นภาษาที่เข้าใจง่ายและบอกตำแหน่งไฟล์ชัดเจน

---

## 🛠️ 1. เครื่องมือที่ต้องมี (เตรียมเครื่อง)
ถ้าคุณย้ายเครื่อง หรือไปทำเครื่องใหม่ ต้องลง 2 โปรแกรมนี้ก่อนเสมอ:
1.  **VS Code** (โปรแกรมแก้ไขโค้ด): [ดาวน์โหลดที่นี่](https://code.visualstudio.com/)
2.  **Node.js** (ตัวรันระบบ): [ดาวน์โหลดที่นี่ (เลือกตัว LTS)](https://nodejs.org/)

---

## ▶️ 2. วิธีรันโปรเจกต์ (เปิดดูหน้าเว็บในเครื่อง)
1.  เปิดโปรแกรม **VS Code**
2.  ไปที่เมนู **File > Open Folder** แล้วเลือกโฟลเดอร์โปรเจกต์นี้
3.  กดเมนู **Terminal > New Terminal** (จะมีหน้าต่างสีดำๆ เด้งขึ้นมาข้างล่าง)
4.  พิมพ์คำสั่ง: `npm install` (เฉพาะครั้งแรก หรือย้ายเครื่องใหม่)
5.  รอจนเสร็จ แล้วพิมพ์: `npm run dev`
6.  กดลิงก์ที่ขึ้นมา (เช่น `http://localhost:5173`) เพื่อดูหน้าเว็บ

---

## ✏️ 3. อยากแก้...ต้องไปไฟล์ไหน?

### 🅰️ อยากแก้ "คำพูด / ข้อความ" บนหน้าเว็บ
ระบบนี้รองรับ 2 ภาษา (ไทย/อังกฤษ) คำศัพท์ทั้งหมดจะถูกเก็บรวมไว้ที่เดียวครับ
*   **ไปที่ไฟล์:** `src/translations/th.ts` (สำหรับภาษาไทย)
*   **ไปที่ไฟล์:** `src/translations/en.ts` (สำหรับภาษาอังกฤษ)
*   **วิธีแก้:** กด `Ctrl` + `F` แล้วพิมพ์คำที่อยากแก้ (เช่น "ลงทะเบียนเคลม") พอเจอแล้วก็แก้ข้อความในเครื่องหมาย `'...'` ได้เลย (ระวังอย่าลบเครื่องหมายจุลภาค `,` ด้านหลังทิ้ง)

### 📋 อยากเพิ่ม/ลบ "ตัวเลือก" ใน Dropdown (ถ้าแก้ผ่านหน้า Admin ไม่ได้)
บางตัวเลือกอาจจะฝังอยู่ในโค้ด เช่น สถานะ หรือ ตัวเลือกบางอย่าง
*   **ไปที่ไฟล์:** `src/constants/options.ts` (ถ้ามี) หรือ `services/mockDb.ts` หรือ `pages/TrackClaim.tsx`
*   **คำแนะนำ:** ถ้าหาไม่เจอ ให้ลองกดค้นหาทั้งโปรเจกต์ (รูปแว่นขยายซ้ายมือ) แล้วพิมพ์คำที่อยากหาครับ

### 🎨 อยากเปลี่ยน "โลโก้" หรือ "สีหลัก"
*   **โลโก้:** ไฟล์รูปภาพจะอยู่ในโฟลเดอร์ `public/`
    *   เปลี่ยนไฟล์ชื่อ `logo.png` หรือ `favicon.ico` (ต้องตั้งชื่อให้เหมือนเดิมเป๊ะๆ แล้วลบไฟล์เก่าออก)
*   **สีของเว็บ:** ส่วนใหญ่จะแก้ที่ไฟล์ `src/index.css` หรือในโค้ดแต่ละหน้า (อันนี้อาจจะต้องระวังหน่อยครับ แนะนำให้ถาม AI ถ้าจะแก้เยอะ)

### 👤 อยากตั้งค่า "Admin" (ใครเป็นแอดมินบ้าง)
ตอนนี้ระบบตั้งค่าให้ `support@sectechnology.co.th` เป็น Super Admin เสมอ
*   ถ้าอยากเพิ่มคนอื่นแบบถาวรในโค้ด:
    *   **ไปที่ไฟล์:** `services/mockDb.ts`
    *   **ค้นหาบรรทัด:** `user.email === 'support@sectechnology.co.th'`
    *   **วิธีแก้:** เพิ่มอีเมลอื่นเข้าไป เช่น `|| user.email === 'boss@sec.com'`

## ☁️ 4. วิธีเชื่อมต่อฐานข้อมูลออนไลน์ (Firebase)
(เพื่อให้ข้อมูลไม่หาย และใช้งานได้หลายเครื่องพร้อมกัน)

ปกติถ้าเราไม่เชื่อม Firebase ระบบจะจำลองข้อมูลขึ้นมาเอง แต่พอปิดเว็บข้อมูลก็หาย
ถ้าอยากให้ข้อมูลอยู่ถาวร ต้องทำตามนี้ครับ (ทำครั้งเดียวจบ):

### ขั้นตอนที่ 1: สร้างโปรเจกต์
1. เข้าเว็บ [Firebase Console](https://console.firebase.google.com/) แล้วล็อกอินด้วย Gmail
2. กด **Create a project** > ตั้งชื่อ (เช่น `my-sec-claim`) > กด Continue ไปเรื่อยๆ จนเสร็จ

### ขั้นตอนที่ 2: เปิดใช้งาน Database
1. เมนูซ้าย เลือก **Build > Firestore Database**
2. กด **Create database**
3. เลือก Server: `asia-southeast1` (สิงคโปร์) จะได้เร็วๆ
4. เลือก **Start in test mode** > กด Create

### ขั้นตอนที่ 3: เปิดใช้งาน Login
1. เมนูซ้าย เลือก **Build > Authentication**
2. กด **Get started**
3. เลือก **Email/Password** > กด **Enable** > Save

### ขั้นตอนที่ 4: รับกุญแจ (API Key) มาใส่ในโค้ด
1. กดที่รูปเฟือง ⚙️ (Project settings) ข้างบนซ้าย
2. เลื่อนลงมาล่างสุด กดไอคอน `</>` (Web)
3. ตั้งชื่อแอพ (เช่น Web) > กด Register app
4. คุณจะเห็นโค้ดภาษาอังกฤษเยอะๆ ให้หาดูค่าพวกนี้:
   - `apiKey: "..."`
   - `authDomain: "..."`
   - `projectId: "..."`
   ... (และตัวอื่นๆ)

### ขั้นตอนที่ 5: เอามาใส่ในไฟล์ .env
1. กลับมาที่ VS Code
2. ไฟล์ชื่อ `.env` (อยู่ในโฟลเดอร์นอกสุด)
3. เอาค่าที่ได้มาใส่แทนที่ `xxxx` เดิม:
   ```env
   VITE_FIREBASE_API_KEY=ค่าที่ได้จาก apiKey
   VITE_FIREBASE_AUTH_DOMAIN=ค่าที่ได้จาก authDomain
   VITE_FIREBASE_PROJECT_ID=ค่าที่ได้จาก projectId
   VITE_FIREBASE_STORAGE_BUCKET=ค่าที่ได้จาก storageBucket
   VITE_FIREBASE_MESSAGING_SENDER_ID=ค่าที่ได้จาก messagingSenderId
   VITE_FIREBASE_APP_ID=ค่าที่ได้จาก appId
   ```
4. **บันทึกไฟล์** แล้วกดปิด/เปิด Terminal ใหม่ (หรือกด Ctrl+C แล้ว `npm run dev` ใหม่)
5. ถ้ารันแล้วขึ้นว่า `🚀 SEC-CLAIM: Connected to Firebase Cloud` แปลว่าเชื่อมต่อสำเร็จ!

## 🚀 5. วิธีอัปเดตขึ้นเว็บจริง (Deploy) (Windows & Mac)
เมื่อแก้ไขเสร็จแล้ว และทดสอบในเครื่อง (`npm run dev`) จนพอใจ ให้ทำดังนี้เพื่อเอาขึ้นเว็บจริง:

1.  เปิด Terminal ใน VS Code (ถ้าเปิดอยู่แล้วกด `Ctrl` + `C` เพื่อหยุดตัวเก่าก่อน)
2.  พิมพ์คำสั่งศักดิ์สิทธิ์:
    ```bash
    npm run build && firebase deploy
    ```
3.  รอจนมันขึ้นว่า **Deploy complete!**
4.  จบครับ! ลูกค้าทั่วโลกจะเห็นหน้าเว็บแบบใหม่ทันที

## 🚚 6. วิธีย้ายไปทำเครื่องอื่น (ต้องตั้งค่าใหม่ไหม?)
**คำตอบ:** ไม่ต้องสร้าง Firebase ใหม่ครับ แค่ทำตามนี้:

1. **Copy โฟลเดอร์โปรเจกต์นี้** ไปใส่เครื่องใหม่ (หรือโหลดจาก Git/Google Drive)
2. **สำคัญ:** เช็คว่ามีไฟล์ `.env` ติดมาด้วยไหม?
   - **ถ้ามี:** จบเลย ใช้งานได้ทันที (ข้ามไปข้อ 4)
   - **ถ้าไม่มี:** (บางที Copy แล้วไฟล์หาย) ให้สร้างไฟล์ `.env` ใหม่ แล้วไปเอา "รหัส" จากเครื่องเดิม หรือจาก Firebase Console (ตามหัวข้อที่ 4) มาใส่
3. ในเครื่องใหม่ ลงโปรแกรม **VS Code** และ **Node.js** ให้เรียบร้อย
4. เปิด VS Code > Terminal (จอดำ)
5. พิมพ์คำสั่ง: `npm install` (เพื่อดึงไลบรารี่ต่างๆ ลงเครื่องใหม่)
6. พิมพ์คำสั่ง: `npm run dev` (เพื่อเริ่มทำงาน)

**สรุป:** สิ่งเดียวที่สำคัญคือไฟล์ `.env` ครับ ถ้าไฟล์นี้อยู่ ก็เชื่อมต่อได้เลย!
