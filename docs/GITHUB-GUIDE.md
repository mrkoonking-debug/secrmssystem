# คู่มือการใช้งาน GitHub แบบ 2 เครื่อง (สำหรับใช้งานสลับเครื่อง)

คู่มือนี้สำหรับเก็บไว้อ่านเตือนความจำ ว่าถ้าย้ายไปทำคอมพิวเตอร์เครื่องอื่นต้องรันคำสั่งอะไร และเวลาหยุดทำงานต้องเซฟโค้ดขึ้นระบบอย่างไร เพื่อให้สลับไปมาได้โดยโค้ดไม่พังและไม่ชนกันครับ

---

## 💻 1. การดึงโปรเจกต์ลงคอมเครื่องใหม่ (ทำครั้งแรกรอบเดียวพอ)

ลงโปรแกรม Git: โหลดที่ git-scm.com (Next จนจบ)
ลงโปรแกรม Node.js: โหลดที่ nodejs.org (เลือกรุ่น LTS)
ลงโปรแกรม VSCode และเปิดโปรแกรมขึ้นมา

1. เปิด VSCode แล้วเปิด Terminal (เมนู `Terminal` -> `New Terminal`)
2. วางคำสั่งเพื่อดึงโปรเจกต์จาก GitHub คลังของคุณ:
   
   git clone https://github.com/mrkoonking-debug/secrmssystem.git
   ```
3. ย้ายเข้าไปในโฟลเดอร์ที่เพิ่งโหลดมาด้วยคำสั่ง:
   
   cd secrmssystem
   ```
4. ติดตั้งเครื่องมือและแพ็กเกจที่จำเป็นสำหรับการเขียนโค้ดทั้งหมด (เหมือนที่คุณเคยรัน `npm install` ไป):
   
   npm install
   ```
5. ทดลองรันระบบเพื่อให้แน่ใจว่าโปรเจกต์พร้อมใช้แล้ว:
   
   npm run dev
   ```

---

## 🔄 2. กฎทอง 2 ข้อสำหรับการทำงานสลับเครื่อง (ต้องทำเป็นประจำ!)

เพื่อป้องกันโค้ดหายและโค้ดทับกัน ให้จำเคล็ดลับสั้นๆ ว่า **"ดึงก่อนเริ่ม ดันก่อนเลิก"** ดังนี้ครับ:

### 🟢 กฎข้อที่ 1: "ดึงงานล่าสุดก่อนทำงาน" (ก่อนเริ่มโค้ดทุกวัน)
เปิดคอมขึ้นมาและเปิดโปรเจกต์นี้ใน VSCode **หน้าต่าง Terminal สิ่งแรกที่ต้องพิมพ์ก่อนเขียนโค้ดทุกวัน** คือพิมพ์:

git pull
```
*(ระบบจะไปดูบนเซิร์ฟเวอร์ ถ้ามีอัปเดตใหม่ๆ จากเครื่องอื่น มันจะโหลดโค้ดใหม่มาเติมฝั่งเราให้ตรงกันทันที โค้ดจะได้ไม่ชนกัน)*

### 🔴 กฎข้อที่ 2: "เซฟและดันขึ้นคลังก่อนเลิกงาน" (ทำเมื่อเขียนโค้ดในแต่ละวันเสร็จ)
เมื่อเขียนโค้ดเสร็จแล้ว ต้องการหยุดงานและไปทำเครื่องอื่น ให้พิมพ์ 3 คำสั่งตามลำดับต่อไปนี้ เพื่อเซฟและดันงานขึ้น GitHub:

git add .

git commit -m "อัปเดตงาน: [ใส่สิ่งที่คุณเพิ่งทำไปสั้นๆ เช่น แก้ไขหน้า Dashboard เปลี่ยนสีปุ่ม บลาๆ]"

git push

*(รอจนสถานะใน Terminal โหลด 100% เรียบร้อย แล้วจึงปิดโปรแกรม VSCode หรือปิดคอมได้ครับ)*

---

### 💡 วัฏจักรการทำงานแบบเซียนๆ
- ตื่นมาเปิดคอม **เครื่อง A** -> พิมพ์ `git pull` -> ทำระบบ -> ทำเสร็จแล้วพิมพ์ `git add .`, `git commit -m "..."`, `git push` -> ไปนอน 🛏️
- วันต่อมาไปทำธุระข้างนอก ใช้ **เครื่อง B** -> พิมพ์ `git pull` -> ทำระบบ -> ทำเสร็จแล้วพิมพ์ `git add .`, `git commit -m "..."`, `git push` -> กลับบ้าน 🚗
- **วนลูปตามนี้ โค้ดจะปลอดภัยสุดๆ และเหมือนกันเป๊ะ 100% ทั้ง 2 เครื่องครับ!**
