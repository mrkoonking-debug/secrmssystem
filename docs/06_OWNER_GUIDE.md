# 📖 คู่มือเจ้าของระบบ (Owner Guide)

คู่มือนี้เขียนสำหรับเจ้าของหรือผู้ดูแลระบบที่ **ไม่ได้เป็นโปรแกรมเมอร์** — ไม่ต้องรู้โค้ดก็แก้ได้ในจุดที่บอก

---

## 🛠️ 1. เตรียมเครื่องให้พร้อม (ทำครั้งเดียว)

ต้องติดตั้งโปรแกรมเหล่านี้ก่อน:

| โปรแกรม | ลิงก์ | หมายเหตุ |
| :--- | :--- | :--- |
| **Node.js** | [nodejs.org](https://nodejs.org) | เลือกตัว **LTS** เท่านั้น |
| **VS Code** | [code.visualstudio.com](https://code.visualstudio.com) | โปรแกรมแก้ไขโค้ด |

---

## ▶️ 2. เปิดโปรเจกต์และรันทดสอบ

1. เปิด **VS Code** → **File → Open Folder** → เลือกโฟลเดอร์ `sec-claim-system`
2. กด **Terminal → New Terminal** (หน้าต่างสีดำโผล่ขึ้นมา)
3. ครั้งแรกให้รัน:
   ```bash
   npm install
   ```
4. รันทุกครั้งหลังจากนั้น:
   ```bash
   npm run dev
   ```
5. กดลิงก์ `http://localhost:5173` ที่ขึ้นมาเพื่อดูหน้าเว็บในเครื่อง

---

## ✍️ 3. สิ่งที่แก้ได้เองโดยไม่ต้องรู้โค้ด

### 3.1 เปลี่ยนโลโก้บริษัท
1. เตรียมไฟล์รูปโลโก้ใหม่ (แนะนำเป็น `.png` พื้นหลังโปร่งใส)
2. ตั้งชื่อไฟล์ว่า **`logo.png`** เป็ะๆ
3. Copy ไฟล์ไปแทนที่ตรงนี้: `public/logo.png`
4. รัน `npm run dev` ใหม่ โลโก้จะเปลี่ยนทันที

### 3.2 ตั้งค่าข้อมูลบริษัท (ชื่อ, ที่อยู่, เลขภาษี)
- Login เข้าระบบ Admin → ไปที่เมนู **Settings** → แก้ไขได้เลย
- ข้อมูลนี้จะถูกดึงไปแสดงในใบพิมพ์เอกสารอัตโนมัติ

### 3.3 เพิ่ม/ลบยี่ห้อสินค้า
- Login เข้าระบบ Admin → ไปที่เมนู **Brands** → เพิ่ม/ลบได้เลย

### 3.4 เพิ่ม/ลบ Distributor (ผู้นำเข้า)
- Login เข้าระบบ Admin → ไปที่เมนู **Distributors** → เพิ่ม/ลบได้เลย

### 3.5 เพิ่ม Staff Account ใหม่
- Login เข้าระบบ Admin → ไปที่เมนู **Users** → กด **Add Staff**
- กรอก Email, Password, ชื่อ, ทีม → กด Save

---

## 🔤 4. แก้ข้อความบนหน้าเว็บ (ต้องแก้ในโค้ด)

1. เปิดไฟล์: `i18n/translations.ts` ใน VS Code
2. กด `Ctrl+F` แล้วพิมพ์คำที่อยากหา (เช่น `ลงทะเบียนเคลม`)
3. แก้ข้อความในเครื่องหมาย `'...'` ได้เลย

> ⚠️ **ระวัง:** อย่าลบหรือขยับเครื่องหมาย `,` `:` `'` `{` `}` เพราะจะทำให้ระบบเด้ง Error

หลังแก้เสร็จให้รัน deploy ใหม่ (ดูหัวข้อ 6)

---

## 🚀 5. กลับมาทำที่เครื่องใหม่ (ย้ายเครื่อง)

> ❓ **ต้องสร้าง Firebase ใหม่ไหม?** → **ไม่ต้องครับ** Firebase อยู่บน Cloud ตลอด

**ทำแค่นี้:**
1. Copy โฟลเดอร์โปรเจกต์ไปเครื่องใหม่
2. ตรวจว่ามีไฟล์ `.env` ติดมาด้วย (สำคัญมาก!)
3. ติดตั้ง Node.js และ VS Code ในเครื่องใหม่
4. เปิด Terminal แล้วรัน:
   ```bash
   npm install
   npm run dev
   ```

> 💡 ถ้าไม่มีไฟล์ `.env` ดูวิธีสร้างใหม่ได้ที่ [02_FIREBASE_SETUP.md](./02_FIREBASE_SETUP.md)

---

## 🌐 6. อัปเดตเว็บจริง (Deploy)

เมื่อแก้ไขเสร็จแล้วและทดสอบในเครื่องว่า OK แล้ว ให้รันคำสั่งนี้:

```bash
npm run build && npx firebase deploy
```

รอจนขึ้นว่า **`Deploy complete!`** → เว็บจริงจะอัปเดตทันที

> ดูคำสั่งทั้งหมดและการแก้ปัญหาได้ที่ [03_DEPLOYMENT.md](./03_DEPLOYMENT.md)

---

## 🔑 7. ลืม Password ของ Admin / Staff

ใช้วิธีนี้:
1. เปิด [Firebase Console](https://console.firebase.google.com/)
2. ไปที่ **Authentication → Users**
3. หา Email ของคนที่ลืมรหัส → กดจุดสามจุด → **Reset password**
4. Firebase จะส่ง Email รีเซ็ตให้อัตโนมัติ

---

## 🆘 8. ถ้าหน้าเว็บเด้ง Error (หน้าดำ / หน้าขาว)

**วิธีตรวจสอบเบื้องต้น:**
1. เปิด Developer Tools (กด F12)
2. ไปแท็บ **Console**
3. ดู Error ที่ขึ้นสีแดง แล้วส่งให้ทีมช่วยดูได้เลย

**สาเหตุที่พบบ่อย:**
- ค่าในไฟล์ `.env` ผิด / หาย
- แก้โค้ดแล้วทำ syntax ผิดพลาด
- Firebase down ชั่วคราว (รอสักครู่แล้วโหลดใหม่)

---

## 📞 9. เมื่อไหรที่ควรโทรหาโปรแกรมเมอร์

| สถานการณ์ | ต้องเรียกโปรแกรมเมอร์? |
| :--- | :--- |
| เปลี่ยนโลโก้ | ❌ ทำเองได้ |
| แก้ข้อความ/คำพูดบนเว็บ | ⚠️ ทำได้ แต่ระวัง syntax |
| เพิ่ม Staff / ยี่ห้อ / Distributor | ❌ ทำเองผ่าน Admin ได้ |
| Deploy ขึ้นเว็บ | ⚠️ ต้องรันคำสั่ง (ดูได้จากคู่มือ) |
| เพิ่มฟีเจอร์ใหม่ | ✅ ต้องให้โปรแกรมเมอร์ทำ |
| เปลี่ยนโครงสร้างฐานข้อมูล | ✅ ต้องให้โปรแกรมเมอร์ทำ |
| เว็บพัง / Error ที่แก้ไม่ได้ | ✅ ต้องให้โปรแกรมเมอร์ดู |
