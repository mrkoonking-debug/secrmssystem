# 📘 คู่มือการใช้งานระบบ SEC RMA (User Manual)

คู่มือนี้จัดทำขึ้นเพื่อให้เจ้าหน้าที่ (Staff), ผู้ดูแลระบบ (Admin) และผู้เกี่ยวข้องในกระบวนการเคลมสินค้า ทราบถึงขั้นตอนการใช้งานระบบ SEC RMA System อย่างละเอียดในแต่ละฟีเจอร์หลัก

---

## 🚀 1. การเข้าสู่ระบบ (Login)
1. ไปที่เว็บไซต์ของระบบ (เช็คลิงก์ที่ใช้งานจริงกับทางผู้ดูแลระบบ เช่น `https://secrmssystem.web.app`)
2. กรอก **Email** และ **Password** ที่ได้รับแจ้ง
3. กดปุ่ม **Sign In** เพื่อเข้าสู่หน้าแดชบอร์ดหลัก

---

## 📊 2. หน้าแดชบอร์ด (Dashboard)
เมื่อเข้าสู่ระบบ หน้าแรกที่จะพบคือ Dashboard ไว้สำหรับดูสรุปภาพรวมและงานที่เร่งด่วน:
- **แถบกรองข้อมูลด้านบน (Filter):** สามารถกรองข้อมูลงานเคลมตามทีมที่รับผิดชอบได้ เช่น `Team A`, `Team B`, `Team C (Network, UPS, Online)`
- **Pending Action:** สรุปจำนวนงานเคลมที่คงค้างและรอการดำเนินการ
- **Aging Analysis (กราฟอายุงาน):** แสดงข้อมูลว่างานเคลมส่วนใหญ่อยู่ในระบบมากี่วัน (Fresh, Aging, Stale) เพื่อให้ติดตามงานเก่าได้ง่าย
- **Urgent Attention:** รายการงานเคลมที่ **"เร่งด่วน"** สีแดงกระพริบ (เช่น ค้างนานเกินกำหนด หรือ รออะไหล่นาน) สามารถคลิกที่กล่องเพื่อเข้าไปประมวลผลงานนั้นได้ทันที

---

## 📝 3. การรับเรื่อง / เปิดบิลงานเคลม (Submit Claim)
แบ่งการเปิดงานเคลมออกเป็น 2 รูปแบบหลัก คือ **เปิดบิลโดยเจ้าหน้าที่** และ **ลูกค้าส่งเรื่องเข้ามาเอง**

### 3.1 สำหรับเจ้าหน้าที่เป็นผู้เปิดบิล (เมนู Submit Claim)
1. ไปที่เมนู **Submit Claim** ในแถบเมนูด้านซ้าย
2. **ข้อมูลลูกค้า (Contact Info):** 
   - กรอก **รหัสอ้างอิง (Quotation No.)** (ถ้ามี)
   - กรอก **ชื่อบริษัท (Company Name)**, **ชื่อผู้ติดต่อ**, และ **เบอร์โทรศัพท์**
   - สำคัญ: เลือก **LINE Account** (ช่องทางที่ลูกค้าติดต่อมา) เพื่อใช้สรุปข้อมูลจัดส่ง
   - กรอก **ที่อยู่สำหรับส่งคืน (Return Address)**
3. **ข้อมูลสินค้า (Product Info):**
   - ค้นหายี่ห้อ (Brand) และกรอกรุ่น (Model), ซีเรียลนัมเบอร์ (Serial Number)
   - เลือก **บริษัทผู้นำเข้า (Distributor)** ที่จะเป็นผู้รับซ่อมต่อ (เช่น VSTECS, SYNNEX)
   - ระบุ **อาการเสีย (Issue Description)** อย่างละเอียด
   - เลือก **อุปกรณ์ที่แนบมาด้วย (Accessories)** เช่น อะแดปเตอร์, กล่อง, เมาส์ เพื่อป้องกันการสูญหาย
   - หากมีหลายชิ้น สามารถกด **+ Add to List** เพื่อเพิ่มสินค้าชิ้นต่อไปในบิลเดียวกันได้เลย
4. เมื่อข้อมูลครบถ้วน กดปุ่ม **Submit (ส่งข้อมูล)** ระบบจะออกเลขบิลให้และสามารถ **พิมพ์ใบแปะกล่อง (Print Label)** ได้

### 3.2 สำหรับลูกค้าส่งเรื่องเอง (Customer Public Form)
1. ลูกค้าจะเข้าเว็บผ่านลิงก์ฟอร์มสำหรับลูกค้า (มักเชื่อมจาก Rich Menu ใน LINE OA)
2. ลูกค้าจะกรอกเบอร์โทรศัพท์, ชื่อ, ที่อยู่จัดส่งกลับ และเลือกประเภท LINE ที่คุยอยู่
3. ลูกค้ากรอกข้อมูลสินค้า, อาการเสีย, และระบุอุปกรณ์ที่ส่งมาพร้อมกัน
4. เมื่อกดยืนยัน ระบบจะแสดง **รหัสอ้างอิง** ให้ลูกค้า และลูกค้าสามารถพิมพ์ใบจ่าหน้ากล่อง (Sender Label) สำหรับแปะกล่องพัสดุส่งมาที่บริษัทได้
5. **รายการของลูกค้าจะเข้าไปรอเจ้าหน้าที่ที่เมนู Incoming Claims**

---

## 📥 4. การจัดการรายการเคลมเข้าใหม่ (Incoming Claims)
เมื่อลูกค้ากรอกฟอร์มเคลมสำเร็จ งานจะเข้ามาอยู่ในหน้านี้
1. ไปที่เมนู **Incoming Claims** 
2. เจ้าหน้าที่จะเห็นรายการที่ลูกค้าเปิดบิลเข้ามาทั้งหมด (สถานะ `PENDING_STAFF`)
3. เมื่อขนส่งมาส่งของที่บริษัท เจ้าหน้าที่จะนำพัสดุมาเทียบกับบิลในหน้านี้
4. หากตรวจสอบว่าของตรงกัน ให้กดปุ่ม **Mark as Received (รับของเข้าระบบแล้ว)** และ **มอบหมายทีมที่รับผิดชอบ**
5. หากข้อมูลผิดพลาด หรือของที่ส่งมาไม่ตรง สามารถกด **Edit (แก้ไข)** ได้

---

## 🔍 5. การค้นหาและดูรายการเคลมทั้งหมด (Claims List)
1. ไปที่เมนู **Claims List** จะพบรายการเคลมทุกสถานะในระบบ
2. **การค้นหา:** พิมพ์รหัสบิล (Ref), Serial Number, ชื่อบริษัท, หรือเบอร์โทรศัพท์ ในช่อง Search ด้านบน
3. **การกรอง (Filter):** สามารถกรองตาม **ทีมที่รับผิดชอบ**, **สถานะงานซ่อม** หรือ **ยี่ห้อ (Brand)**
4. คลิกที่รหัสบิล หรือ สินค้าที่ต้องการ เพื่อเข้าไปดูรายละเอียด (Job Detail)

---

## ⚙️ 6. การอัปเดตสถานะงานซ่อม (Job Detail & Update Status)
เมื่อคลิกเข้ามาดูงานเคลมแต่ละตัว (Job Detail) จะมีเครื่องมือสำหรับอัปเดตงาน:

- **1. Update Status (อัปเดตสถานะ):**
  เป็นการเปลี่ยนสถานะงานซ่อมเพื่อให้ลูกค้ารับทราบความคืบหน้า เช่น 
  - *รอรับเข้า (Pending), ตรวจสอบอาการ, ส่งซ่อมศูนย์, รออะไหล่, เสนอราคาลูกค้า, ซ่อมเสร็จแล้ว, ส่งคืนลูกค้า*
  - เมื่ออัปเดตสถานะ ระบบจะบันทึกประวัติไว้ด้านขวามือ (History Timeline) ว่าใครเปลี่ยนสถานะเวลาไหน
- **2. แก้ไขข้อมูลซ่อม (Edit RMA / Update Resolution):**
  สำหรับเปลี่ยนข้อมูลตั้งต้น หรือ **เพิ่มผลการซ่อม (Resolution / Action Taken)** เช่น "เปลี่ยนเมนบอร์ด", "อัปเดตเฟิร์มแวร์" โดยสามารถแนบ **ไฟล์รูปภาพ** ยืนยันให้ลูกค้าดูได้
- **3. เปลี่ยนทีมรับผิดชอบ (Re-assign Team):** หากรับงานมาผิดทีม สามารถส่งต่อไปยังทีมอื่นพร้อมเซ็ตความสำคัญ (Priority) ได้
- **4. พิมพ์เอกสาร (Print Label):** พิมพ์ใบปะหน้าใหม่ได้เรื่อยๆ

---

## 🖨️ 7. การพิมพ์ฟอร์มเคลม (Print Documents)
จากหน้า Job Detail เมื่อคลิกไปที่เมนู **Print & Documents** สามารถเลือกรูปแบบการพิมพ์ได้:
1. **[แบบฟอร์มศูนย์ซ่อม - Distributor Form] (สีส้ม):**
   - เพื่อใช้ แนบไปกับตัวสินค้าเวลาส่งซ่อมไปศูนย์ใหญ่หรือ Distributor
   - มีช่องยืนยัน อุปกรณ์/ของแถมที่แนบไปครบถ้วน (Accessories Checklist) ช่วยป้องกันของหาย
2. **[แบบฟอร์มลูกค้า - Customer Form] (สีฟ้า):**
   - เพื่อใช้ แนบส่งคืนให้ลูกค้า เมื่อสินค้าซ่อมเสร็จแล้ว
   - มีสรุปอาการเสีย และ **ผลการซ่อม/แนวทางแก้ไข (Resolution)** อย่างชัดเจน

---

## 📱 8. ระบบติดตามสถานะสำหรับลูกค้า (Customer Track Claim)
- ลูกค้าสามารถเข้าหน้า **Track Claim Status** ของบริษัทได้ด้วยตัวเอง
- ลูกค้าต้องกรอกรหัส **หมายเลขโทรศัพท์ (Phone Number)** และ **Serial Number** (หรือรหัสอ้างอิง) พร้อมกัน ตัวระบบถึงจะแสดงข้อมูล (ปลอดภัย ไม่รั่วไหล)
- ลูกค้าจะเห็น Timeline ง่ายๆ ทันทีว่าตอนนี้สินค้าระบุสถานะถึงขั้นตอนไหน กำลังซ่อม, ส่งศูนย์นอก, หรือส่งคืนแล้ว

---

## 🛠️ 9. การตั้งค่าระบบสำหรับ Admin (Settings & Management)
*สงวนสิทธิ์เฉพาะผู้ใช้ระดับ Admin เท่านั้น*
1. **User Management:** สำหรับเพิ่ม/ลบ สิทธิ์พนักงาน, กำหนดทีม (Team A, B, C) และเปลี่ยนรหัสผ่านให้ Staff
2. **Brand Management:** สำหรับเพิ่มและจัดการชื่อยี่ห้อสินค้า เพื่อให้พนักงานมีตัวเลือกตอนพิมพ์บิล
3. **Distributor Management:** จัดการรายชื่อบริษัทผู้นำเข้า เช่น VSTECS, SYNNEX
4. **Settings:** อัปเดตข้อมูลบริษัท ที่อยู่ เพื่อนำไปแสดงผลบนหัวเอกสารการพิมพ์อัตโนมัติ
