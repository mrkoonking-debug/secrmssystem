# 📘 คู่มือการใช้งานระบบ SEC RMA (User Manual)

คู่มือนี้จัดทำขึ้นเพื่อให้เจ้าหน้าที่ (Staff), ผู้ดูแลระบบ (Admin) และผู้เกี่ยวข้องในกระบวนการเคลมสินค้า ทราบถึงขั้นตอนการใช้งานระบบ SEC RMA System อย่างละเอียดในแต่ละฟีเจอร์หลัก

---

## 🚀 1. การเข้าสู่ระบบ (Login)
1. ไปที่เว็บไซต์ `https://secrmssystem.web.app`
2. กรอก **Email** และ **Password** ที่ได้รับ
3. กดปุ่ม **Sign In** เพื่อเข้าสู่หน้า Dashboard

---

## 📊 2. หน้าแดชบอร์ด (Dashboard)
เมื่อเข้าสู่ระบบ หน้าแรกจะแสดงสรุปภาพรวม:
- **แถบกรอง (Filter):** กรองข้อมูลตามทีม เช่น Team A (Hikvision), Team B (Dahua), Team C (Network/UPS/Online)
- **Pending Action:** จำนวนงานเคลมที่คงค้างและรอดำเนินการ
- **Aging Analysis:** กราฟแสดงอายุงาน (Fresh / Aging / Stale)
- **Urgent Attention:** รายการงานเร่งด่วน (สีแดงกระพริบ) คลิกเข้าดำเนินการได้ทันที

---

## 📝 3. การรับเรื่อง / เปิดบิลงานเคลม (Submit Claim)

### 3.1 เจ้าหน้าที่เปิดบิล (เมนู Submit Claim)
1. ไปที่เมนู **Submit Claim**
2. **ข้อมูลลูกค้า:** กรอกรหัสอ้างอิง (ถ้ามี), ชื่อบริษัท, ชื่อผู้ติดต่อ, เบอร์โทร, LINE Account, ที่อยู่ส่งคืน
3. **ข้อมูลสินค้า:**
   - เลือกยี่ห้อ (Brand), กรอกรุ่น (Model), Serial Number
   - เลือก Distributor (ผู้นำเข้า)
   - ระบุอาการเสีย
   - **เลือกอุปกรณ์ที่แนบมา (บังคับ):** ต้องเลือกอย่างน้อย 1 รายการ หรือกด "มีแต่ตัวเครื่อง"
4. หากมีหลายชิ้น กด **+ Add to List** เพิ่มสินค้าในบิลเดียวกันได้
5. กด **Submit** → ระบบออกเลข Job ID + พิมพ์ใบแปะกล่องได้

### 3.2 ลูกค้าลงทะเบียนเอง (Customer Form)
1. ลูกค้าเข้าเว็บผ่านลิงก์ (เช่น จาก LINE OA)
2. กรอกข้อมูลตนเอง + สินค้า + อุปกรณ์ที่ส่งมา
3. ระบบแสดงรหัสอ้างอิง + ใบจ่าหน้ากล่อง (Sender Label)
4. **รายการจะเข้าไปรอที่เมนู Incoming Claims** ของเจ้าหน้าที่

---

## 📥 4. การจัดการรายการเคลมเข้าใหม่ (Incoming Claims)
1. ไปที่เมนู **Incoming Claims**
2. เจ้าหน้าที่จะเห็นรายการที่ลูกค้าส่งมาทั้งหมด (สถานะ `PENDING_STAFF`)
3. เมื่อได้รับพัสดุ ให้เทียบของกับบิล → กด **Mark as Received** + **มอบหมายทีม**
4. หากข้อมูลผิดพลาด กด **Edit** แก้ไขได้

---

## 🔍 5. รายการเคลมทั้งหมด (Claims List)
1. ไปที่เมนู **Claims List**
2. **ค้นหา:** พิมพ์รหัส Job/Ref, Serial Number, ชื่อบริษัท, หรือเบอร์โทร
3. **กรอง:** ตามทีม, สถานะ, หรือยี่ห้อ
4. คลิกที่รายการเพื่อเข้าดูรายละเอียด (Job Detail)

---

## ⚙️ 6. การอัปเดตสถานะงานซ่อม (Edit RMA — ขั้นตอนหลัก)

เมื่อเข้าแก้ไขงาน (Edit) ระบบจะแสดง **ขั้นตอนถัดไป** อัตโนมัติตามสถานะปัจจุบัน:

### ขั้นตอนที่ 1: รับเรื่อง → เริ่มตรวจสอบ
- สถานะ: `PENDING` (รอดำเนินการ)
- กดปุ่ม 🔍 **เริ่มตรวจสอบ** → เปลี่ยนเป็น `DIAGNOSING`

### ขั้นตอนที่ 2: ตรวจสอบเสร็จ → เลือกเส้นทาง
- สถานะ: `DIAGNOSING` (กำลังตรวจสอบ)
- ⚠️ **บังคับกรอก "อาการที่พบ" ก่อน** (ถ้ายังไม่รู้ให้ใส่ N/A)
- เลือก 1 ใน 3 ทาง:

| ปุ่ม | ความหมาย | สิ่งที่เกิดขึ้น |
| :--- | :--- | :--- |
| 📦 **ส่งเคลมศูนย์** | ต้องส่งไปซ่อมที่ผู้นำเข้า | เปลี่ยนเป็น `WAITING_PARTS` |
| 🔧 **แก้ไข Config/Firmware** | แก้ที่ร้านได้เลย (จบที่ร้าน) | เปลี่ยนเป็น `REPAIRED` |
| ↩️ **ไม่พบอาการเสีย** | ตรวจแล้วใช้งานปกติ | เปลี่ยนเป็น `REPAIRED` (ไม่ต้องกรอกอาการ) |

### ขั้นตอนที่ 3 (เส้นทางส่งศูนย์): เลือกอุปกรณ์ที่จะส่ง
- เมื่อกดส่งเคลมศูนย์ ต้อง **เลือกอุปกรณ์ที่จะส่งไปศูนย์** อย่างน้อย 1 รายการ
- แยกเลือกได้ว่าจะส่ง "ตัวเครื่อง" อะแดปเตอร์ สาย หรืออุปกรณ์อื่น
- รายการที่ไม่เลือก = เก็บไว้ที่ร้าน

### ขั้นตอนที่ 4 (เส้นทางส่งศูนย์): รับของคืนจากศูนย์
- สถานะ: `WAITING_PARTS` (รออะไหล่/รอศูนย์)
- เมื่อศูนย์ส่งของกลับมา กดปุ่ม 📥 **รับของคืนจากศูนย์**
- Popup จะเปิดขึ้นให้กรอก:
  - **วิธีดำเนินการ** (เปลี่ยนอะไหล่ / เปลี่ยนเครื่อง / อัปเดต Firmware ฯลฯ)
  - **รายละเอียดเพิ่มเติม** (เช่น เปลี่ยน Mainboard)
  - **S/N ตัวใหม่** (ถ้าเปลี่ยนเครื่อง)
  - **เลข RMA Vendor** (ถ้ามี)
- กด **บันทึกผล** → เปลี่ยนเป็น `REPAIRED`

### ขั้นตอนที่ 5: ตรวจสอบและปิดงาน
- สถานะ: `REPAIRED` (ดำเนินการเสร็จ)
- กดปุ่ม ✅ **ตรวจสอบและปิดงาน**
- Popup จะแสดง **สรุปข้อมูลทั้งหมด** ก่อนปิดงาน:
  - สินค้า, อาการ, ผลการดำเนินการ, S/N ใหม่
- ⚠️ **เมื่อปิดงานแล้วข้อมูลจะถูกล็อค** ต้องปลดล็อคก่อนจึงแก้ไขได้
- กด **ยืนยันปิดงาน** → เปลี่ยนเป็น `CLOSED`

> 💡 สามารถกด "เปลี่ยนสถานะด้วยตนเอง" เพื่อเลือกสถานะอื่นนอกเหนือจาก Flow ปกติได้

## 🖨️ 7. การพิมพ์เอกสาร (Print Documents)

จากหน้า Job Detail กดที่เมนูพิมพ์เอกสาร:

### 7.1 ใบส่งเคลมผู้นำเข้า (สีส้ม)
- ใช้แนบไปกับสินค้าเมื่อส่งซ่อมที่ศูนย์
- มี Accessories Checklist ป้องกันของหาย
- ⭐ **ฟีเจอร์สำคัญ: แยกใบอัตโนมัติตาม Distributor**
  - ถ้าบิลเดียวกันมีสินค้าที่ต้องส่ง**ต่าง Distributor** (เช่น ชิ้นหนึ่ง VSTECS อีกชิ้น SYNNEX)
  - ระบบจะ**แยกใบส่งเคลมให้อัตโนมัติ** — ได้ 2 ใบแยกตาม Distributor
  - ไม่ต้องทำใบเอง จัดกลุ่มให้แล้ว

### 7.2 ใบรับคืนสินค้าลูกค้า (สีฟ้า)
- ใช้แนบส่งคืนให้ลูกค้าเมื่อซ่อมเสร็จ
- มีสรุปอาการเสีย + ผลการซ่อม
- ⭐ **ฟีเจอร์สำคัญ: แสดงเฉพาะงานที่เสร็จแล้ว**
  - ถ้าบิลเดียวกันมีสินค้า 2 ชิ้น แต่เสร็จแค่ 1 ชิ้น
  - ใบส่งคืนจะ**แสดงเฉพาะชิ้นที่เสร็จแล้ว** (สถานะ REPAIRED/CLOSED)
  - ชิ้นที่ยังซ่อมไม่เสร็จจะไม่ขึ้นในใบนี้

---

## 📱 8. ระบบติดตามสถานะสำหรับลูกค้า (Customer Track Claim)
- ลูกค้าเข้าหน้า **Track Claim Status** ของบริษัทได้เอง
- ต้องกรอก **เบอร์โทรศัพท์** และ **Serial Number** (หรือรหัสอ้างอิง) พร้อมกัน ระบบถึงจะแสดงข้อมูล
- ลูกค้าจะเห็น Timeline ว่าสินค้าอยู่ขั้นตอนไหน: กำลังตรวจสอบ / ส่งศูนย์ / ซ่อมเสร็จ / ส่งคืนแล้ว

## 🛠️ 9. การตั้งค่าระบบสำหรับ Admin (Settings & Management)
*สงวนสิทธิ์เฉพาะผู้ใช้ระดับ Admin เท่านั้น*

| เมนู | หน้าที่ |
| :--- | :--- |
| **User Management** | เพิ่ม/ลบ Staff, กำหนดทีม, เปลี่ยนรหัสผ่าน |
| **Brand Management** | จัดการรายชื่อยี่ห้อสินค้า |
| **Distributor Management** | จัดการรายชื่อบริษัทผู้นำเข้า |
| **Settings** | อัปเดตข้อมูลบริษัท (ชื่อ/ที่อยู่/โลโก้) สำหรับหัวเอกสาร |

## ⚠️ 10. ระบบ Validation (การแจ้งเตือนอัตโนมัติ)

ระบบมีการตรวจสอบข้อมูลอัตโนมัติ หากข้อมูลไม่ครบจะ**บล็อกไม่ให้ดำเนินการต่อ** พร้อมแจ้งเตือน:

| สถานการณ์ | สิ่งที่เกิดขึ้น |
| :--- | :--- |
| กดส่งเคลมศูนย์/แก้ไข Config โดยไม่กรอก "อาการที่พบ" | ❌ บล็อก + Toast แดง + ช่องขอบแดง + auto-focus ไปที่ช่อง |
| กดบันทึกโดยไม่เลือกอุปกรณ์ที่จะส่งศูนย์ | ❌ บล็อก + Toast แดง |
| กดปิดงานโดยไม่เลือกวิธีดำเนินการ | ❌ บล็อก + Toast แดง |
| ไม่มีข้อมูลเปลี่ยนแปลงแต่กดบันทึก | ℹ️ Toast แจ้งว่าไม่มีอะไรเปลี่ยน |
| กด "ไม่พบอาการเสีย" | ✅ ไม่ต้องกรอกอาการ — กดได้เลย |

> 💡 ถ้ายังไม่ทราบอาการที่แท้จริง ให้ใส่ **N/A** ในช่อง "อาการที่พบ" ไปก่อนได้
