# บทที่ 2
# แนวคิดและทฤษฎีที่เกี่ยวข้อง

การพัฒนาระบบติดตามสถานะและลงทะเบียนเคลมสินค้าผ่านเว็บแอปพลิเคชัน (SEC RMS System) บริษัท เอส อี ซี เทคโนโลยี จำกัด ได้ศึกษาแนวคิด ทฤษฎี และงานที่เกี่ยวข้อง เพื่อนำมาพัฒนาระบบ โดยมีรายละเอียดดังต่อไปนี้

---

## งานวิจัยที่เกี่ยวข้อง

&emsp;&emsp;ณรงค์ ล่ำดี (2560, บทคัดย่อ) ได้ศึกษาและพัฒนาระบบติดตามความก้าวหน้างานวิจัย เผยแพร่ในวารสารวิชาการมหาวิทยาลัยธนบุรี (วิทยาศาสตร์และเทคโนโลยี) ปีที่ 2 ฉบับที่ 3 (กรกฎาคม-กันยายน 2560) หน้า 33-43 ผู้จัดทำพบว่าระบบสามารถอำนวยความสะดวกให้กับผู้ใช้งานในการติดตามความก้าวหน้างานวิจัยได้อย่างมีประสิทธิภาพ โดยนำเทคโนโลยีสารสนเทศเข้าไปช่วยในการจัดการข้อมูลให้สามารถทำการค้นหาและจัดการข้อมูลได้อย่างมีประสิทธิภาพ ระบบติดตามความก้าวหน้างานวิจัย ได้พัฒนาโดย PHP, MySQL ประกอบด้วย 5 โมดูล ได้แก่ ส่วนการเข้าสู่ระบบ ส่วนแสดงข้อมูลส่วนตัว ส่วนติดตามความก้าวหน้างานวิจัย ส่วนเอกสารงานวิจัย และส่วนรายงานความก้าวหน้างานวิจัย ผลที่ได้จากการพัฒนาระบบดังกล่าว มีประโยชน์ต่อผู้ใช้งานที่ทำหน้าที่ดูแลจัดการข้อมูลให้สามารถเพิ่มข้อมูล แก้ไข เปลี่ยนแปลงข้อมูลได้โดยง่าย และเข้าถึงข้อมูลได้โดยง่าย สะดวกรวดเร็วยิ่งขึ้น จากผลการศึกษาพบว่าการพัฒนาระบบติดตามความก้าวหน้างานวิจัย มีประโยชน์ต่อเจ้าหน้าที่ในการเก็บข้อมูล สามารถจัดเก็บ แก้ไข เปลี่ยนแปลงข้อมูลโดยเข้าถึงได้ง่ายและรวดเร็วมากยิ่งขึ้น และจากการประเมินความพึงพอใจจากการใช้งานระบบของผู้ใช้ พบว่าในด้านภาพรวมด้านฟังก์ชันงานของระบบ (Function Test) อยู่ในระดับดี (X̅ = 4.22, SD = 0.72) และในภาพรวมด้านง่ายต่อการใช้งานของระบบ (Usability Test) อยู่ในระดับดี (X̅ = 4.10, SD = 0.68) งานวิจัยนี้มีความเกี่ยวข้องกับระบบ SEC RMS ในแง่ของการพัฒนาระบบติดตามสถานะและความก้าวหน้าผ่านเว็บแอปพลิเคชัน ซึ่งมีแนวคิดที่คล้ายคลึงกับการติดตามสถานะการเคลมสินค้า

&emsp;&emsp;ประชุม พันออด (2564, บทคัดย่อ) ได้ศึกษาและพัฒนาระบบบริหารการจัดการห้องประชุมออนไลน์ด้วยเทคโนโลยีคลาวด์ ของคณะวิทยาศาสตร์และเทคโนโลยี มหาวิทยาลัยราชภัฏเพชรบุรี เผยแพร่ในรายงานสืบเนื่องจากการประชุมวิชาการเสนอผลงานวิจัยระดับชาติด้านวิทยาศาสตร์และเทคโนโลยี มหาวิทยาลัยราชภัฏจันทรเกษม ครั้งที่ 4 หน้า 90-97 ผู้จัดทำพบว่าระบบสามารถอำนวยความสะดวกให้กับผู้ใช้งานเพื่อให้เกิดความสะดวกรวดเร็วในการทำงานและมีประสิทธิภาพมากขึ้น โดยนำเทคโนโลยีคลาวด์เข้าไปช่วยในการจัดการข้อมูลให้สามารถทำการค้นหาและจัดการข้อมูลได้อย่างมีประสิทธิภาพ กลุ่มเป้าหมายคือ บุคลากรมหาวิทยาลัยราชภัฏเพชรบุรีที่เคยใช้งานระบบการขอใช้ห้องประชุม จำนวน 30 คน โดยการเลือกแบบเจาะจง เครื่องมือที่ใช้ในการวิจัยครั้งนี้คือ ระบบบริหารการจัดการห้องประชุมออนไลน์ด้วยเทคโนโลยีคลาวด์และแบบประเมินความพึงพอใจ ผลที่ได้จากการพัฒนาระบบดังกล่าว มีประโยชน์ต่อผู้ใช้งานที่ทำหน้าที่ดูแลจัดการข้อมูลให้สามารถเพิ่มข้อมูล ตรวจสอบวัน/เวลาว่างของห้องประชุม แก้ไข เปลี่ยนแปลงข้อมูลได้โดยง่าย โดยเอกสารคำร้องจัดส่งผ่านอีเมลโดยอัตโนมัติ และมีการแจ้งเตือนผ่านทาง LINE จากผลการศึกษาพบว่าการพัฒนาระบบบริหารการจัดการห้องประชุมออนไลน์ด้วยเทคโนโลยีคลาวด์ มีประโยชน์ต่อเจ้าหน้าที่ในการจัดการข้อมูลห้องประชุม สามารถจัดเก็บ แก้ไข เปลี่ยนแปลงข้อมูลโดยเข้าถึงได้ง่ายและรวดเร็วมากยิ่งขึ้น และจากการประเมินความพึงพอใจจากการใช้งานระบบของผู้ใช้ พบว่าในด้านภาพรวมด้านฟังก์ชันงานของระบบ (Function Test) อยู่ในระดับดีมาก (X̅ = 4.82, SD = 0.41) และในภาพรวมด้านง่ายต่อการใช้งานของระบบ (Usability Test) อยู่ในระดับดีมาก (X̅ = 4.78, SD = 0.38) งานวิจัยนี้มีความเกี่ยวข้องกับระบบ SEC RMS ในแง่ของการนำเทคโนโลยีคลาวด์ (Cloud Technology) มาพัฒนาระบบบริหารจัดการออนไลน์ ซึ่งมีแนวคิดที่คล้ายคลึงกับการใช้ Google Firebase เป็นแพลตฟอร์มคลาวด์สำหรับจัดเก็บข้อมูลและเผยแพร่ระบบ

&emsp;&emsp;ปฐมฤกษ์ บัวแก้ว และ ไชยยันต์ ปาละมาณ (2568, บทคัดย่อ) ได้ศึกษาและพัฒนาระบบแจ้งซ่อมออนไลน์ คณะแพทยศาสตร์ มหาวิทยาลัยสงขลานครินทร์ เผยแพร่ในวารสารส่งเสริมและพัฒนาวิชาการสมัยใหม่ ปีที่ 3 ฉบับที่ 3 (พฤษภาคม-มิถุนายน 2568) หน้า 81-101 ผู้จัดทำพบว่าระบบสามารถอำนวยความสะดวกให้กับผู้ใช้งานเพื่อให้เกิดความสะดวกรวดเร็วในการทำงานและมีประสิทธิภาพมากขึ้น โดยนำเทคโนโลยีสารสนเทศเข้าไปช่วยในการจัดการข้อมูลให้สามารถทำการค้นหาและจัดการข้อมูลได้อย่างมีประสิทธิภาพ ผลที่ได้จากการพัฒนาระบบดังกล่าว มีประโยชน์ต่อผู้ใช้งานที่ทำหน้าที่ดูแลจัดการข้อมูลให้สามารถเพิ่มข้อมูล รายการแจ้งซ่อม ติดตามสถานะการซ่อม แก้ไข เปลี่ยนแปลงข้อมูลได้โดยง่าย และเข้าถึงข้อมูลได้โดยง่าย สะดวกรวดเร็วยิ่งขึ้น จากผลการศึกษาพบว่าการพัฒนาระบบแจ้งซ่อมออนไลน์ คณะแพทยศาสตร์ มหาวิทยาลัยสงขลานครินทร์ มีประโยชน์ต่อเจ้าหน้าที่ในการเก็บข้อมูลการแจ้งซ่อม สามารถจัดเก็บ แก้ไข เปลี่ยนแปลงข้อมูลโดยเข้าถึงได้ง่ายและรวดเร็วมากยิ่งขึ้น และจากการประเมินความพึงพอใจจากการใช้งานระบบของผู้ใช้ พบว่าในด้านภาพรวมด้านฟังก์ชันงานของระบบ (Function Test) อยู่ในระดับมากที่สุด (X̅ = 4.52, SD = 0.61) และในภาพรวมด้านง่ายต่อการใช้งานของระบบ (Usability Test) อยู่ในระดับมาก (X̅ = 4.48, SD = 0.72) งานวิจัยนี้มีความเกี่ยวข้องกับระบบ SEC RMS ในแง่ของการพัฒนาระบบแจ้งซ่อมและติดตามสถานะการซ่อมบำรุงผ่านเว็บแอปพลิเคชัน ซึ่งมีแนวคิดที่คล้ายคลึงกับระบบติดตามสถานะการเคลมสินค้า

## ทฤษฎีที่เกี่ยวข้อง

### 1. E-R Diagram
&emsp;&emsp;E-R Diagram (Entity-Relationship Diagram) คือ แบบจำลองที่ใช้อธิบายโครงสร้างของฐานข้อมูลซึ่งเขียนออกมาในลักษณะของรูปภาพ การอธิบายโครงสร้างและความสัมพันธ์ของข้อมูล (Relationship) ประกอบด้วย เอนทิตี้ (Entity) เป็นวัตถุหรือสิ่งของที่สนใจในระบบงานนั้น ๆ แอททริบิว (Attribute) เป็นคุณสมบัติของวัตถุที่สนใจ ความสัมพันธ์ (Relationship) คือ ความสัมพันธ์ระหว่างเอนทิตี้ E-R Diagram มีความสำคัญต่อการพัฒนาระบบงานฐานข้อมูล Application ต่าง ๆ ที่ต้องการเก็บข้อมูลอย่างมีระบบมีโครงสร้าง ดังนั้น E-R Diagram จึงใช้เพื่อเป็นเอกสารในการสื่อสารระหว่างนักออกแบบระบบ และนักพัฒนาระบบเพื่อให้สื่อสารอย่างตรงกัน และเป็นสากล

### 2. Use Case Diagram
&emsp;&emsp;Use Case Diagram คือ แผนภาพที่แสดงการทำงานของผู้ใช้ระบบ (User) และความสัมพันธ์กับระบบย่อย (Sub Systems) ภายในระบบใหญ่ ในการเขียน Use Case Diagram ผู้ใช้ระบบ (User) จะถูกกำหนดว่าให้เป็น Actor และระบบย่อย (Sub Systems) คือ Use Case จุดประสงค์หลักของการเขียน Use Case Diagram เพื่อเล่าเรื่องราวทั้งหมดของระบบว่ามีการทำงานอะไรบ้าง เป็นการดึง Requirement หรือเรื่องราวต่าง ๆ ของระบบจากผู้ใช้งาน ซึ่งถือว่าเป็นจุดเริ่มต้นในการวิเคราะห์และออกแบบระบบ สัญลักษณ์ที่ใช้ใน Use Case Diagram จะใช้สัญลักษณ์รูปคนแทน Actor ใช้สัญลักษณ์วงรีแทน Use Case และใช้เส้นตรงในการเชื่อม Actor กับ Use Case เพื่อแสดงการใช้งานของ Use Case ของ Actor นอกจากนั้น Use Case ทุก ๆ ตัวจะต้องอยู่ภายในสี่เหลี่ยมเดียวกันซึ่งมีชื่อของระบบระบุอยู่

### 3. Sequence Diagram
&emsp;&emsp;Sequence Diagram คือ แผนภาพที่แสดงให้เห็นถึงการปฏิสัมพันธ์ (Interaction) ระหว่างอ็อบเจ็กต์ของคลาส โดยเฉพาะการส่ง Message ระหว่างอ็อบเจ็กต์ตามลำดับของเวลา (Sequence) ที่เกิดเหตุการณ์ขึ้นจากน้อยไปมาก โดยจะมีสัญลักษณ์แสดงให้เห็นลำดับของการส่ง Message ตามเวลาส่งอย่างชัดเจน

### 4. Data Table
&emsp;&emsp;Data Table คือ กลุ่มของข้อมูลที่ถูกจัดเก็บไว้ในชุดเดียวกัน โดยเปรียบเหมือนแผ่นตาราง 1 ตารางที่แบ่งเป็น Column (แนวตั้ง) และ Rows (แนวนอน) ที่ถูกจัดเก็บไว้เพียง 1 Table เท่านั้น Data Table สามารถแปลงมาจาก Data Set หรือ Data Adapter หรือจะเป็นการสร้างชุดข้อมูล Data Table ขึ้นมาเองก็ได้

### 5. Gantt Chart
&emsp;&emsp;Gantt Chart คือ แผนผังกำหนดงานมักใช้ในด้านการจัดการโครงการต่าง ๆ ในองค์การขนาดใหญ่ ซึ่งอาจมีขั้นตอนซับซ้อน และมากมาย โดยจะใช้เป็นเทคนิคเครื่องมือช่วยการปฏิบัติงานของผู้บริหาร ในการดำเนินการแก้ไข การควบคุม การวางแผนที่เหมาะสม เพื่อช่วยสนับสนุนให้การดำเนินงานบรรลุเป้าหมาย และวัตถุประสงค์ของโครงการอย่างมีประสิทธิภาพ ผังในลักษณะนี้จะแสดงถึงปริมาณงานและกำหนดเวลาที่จะต้องใช้ เพื่อทำงานนั้นให้ลุล่วง เป็นแผนภูมิในรูปของกราฟแท่งที่ประกอบด้วย แกนหลัก 2 แกน คือ แกนนอนแสดงถึงเวลาในการทำงานตลอดโครงการ และแกนตั้งแสดงถึงงานหรือกิจกรรมที่ต้องทำ แท่งกราฟวางตัวในแนวนอน ความยาวของแท่งกราฟเป็นสัดส่วนโดยตรงกับระยะเวลาในการทำงาน

## ภาษาที่ใช้ในพัฒนาระบบ

### 1. HTML (HyperText Markup Language)
&emsp;&emsp;HTML หรือ HyperText Markup Language คือ ภาษาคอมพิวเตอร์รูปแบบหนึ่งที่มีลักษณะเป็นภาษาในเชิงการบรรยายเอกสารไฮเปอร์มีเดีย (Hypermedia Document Description Language) เพื่อนำเสนอเอกสารนั้นเผยแพร่ในระบบเครือข่ายอินเทอร์เน็ต มีโครงสร้างการเขียนที่อาศัยตัวกำกับ เรียกว่า แท็ก (Tag) ควบคุมการแสดงผลของข้อความ รูปภาพ หรือวัตถุอื่น ๆ ผ่านโปรแกรมเว็บเบราว์เซอร์ (Web Browser)

### 2. CSS (Cascading Style Sheets)
&emsp;&emsp;CSS หรือ Cascading Style Sheets คือ ภาษาคอมพิวเตอร์ภาษาหนึ่งที่ใช้ในการจัดรูปแบบและโครงสร้างของเอกสารที่เขียนจากภาษา HTML โดยภาษา CSS นั้นสามารถใช้งานได้หลากหลาย และมีความยืดหยุ่นสามารถใช้งานกับภาษา XML, SVG และ XUL

### 3. JavaScript
&emsp;&emsp;JavaScript คือ ภาษาคอมพิวเตอร์สำหรับการเขียนโปรแกรมบนระบบอินเทอร์เน็ตที่ได้รับความนิยมอย่างสูง เป็นภาษาสคริปต์เชิงวัตถุ (Object-Oriented Scripting Language) ซึ่งในการสร้างและพัฒนาเว็บไซต์ (ใช้ร่วมกับ HTML) เพื่อให้เว็บไซต์มีการเคลื่อนไหว สามารถตอบสนองผู้ใช้งานได้มากขึ้น มีวิธีการทำงานในลักษณะ "แปลความและดำเนินงานไปทีละคำสั่ง" (Interpret) สามารถทำงานข้ามแพลตฟอร์มได้ โดยทำงานร่วมกับภาษา HTML ได้ทั้งทางฝั่งไคลเอนต์ (Client) และทางฝั่งเซิร์ฟเวอร์ (Server)

### 4. TypeScript
&emsp;&emsp;TypeScript คือ ภาษาโปรแกรมที่พัฒนาโดย Microsoft เป็นภาษาที่ต่อยอดมาจาก JavaScript โดยเพิ่มระบบ Type System เข้ามา ทำให้สามารถกำหนดชนิดข้อมูล (Type) ให้กับตัวแปร พารามิเตอร์ของฟังก์ชัน และค่าที่ส่งกลับได้อย่างชัดเจน ช่วยลดข้อผิดพลาด (Bug) ที่อาจเกิดขึ้นในระหว่างการพัฒนา เนื่องจาก TypeScript จะตรวจสอบข้อผิดพลาดตั้งแต่ขั้นตอนการเขียนโค้ด (Compile-time) ก่อนที่จะนำไปใช้งานจริง TypeScript ได้รับความนิยมอย่างแพร่หลายในการพัฒนาเว็บแอปพลิเคชันขนาดกลางถึงขนาดใหญ่ เนื่องจากช่วยให้โค้ดมีความเป็นระเบียบ อ่านง่าย และบำรุงรักษาง่ายมากยิ่งขึ้น (Microsoft, 2024, https://www.typescriptlang.org/)

## เทคโนโลยีและเฟรมเวิร์กที่ใช้ในการพัฒนาระบบ

### 1. React
&emsp;&emsp;React คือ JavaScript Library สำหรับการสร้างส่วนติดต่อผู้ใช้ (User Interface) ที่พัฒนาโดย Meta (Facebook) เปิดตัวครั้งแรกในปี ค.ศ. 2013 React ใช้แนวคิดของ Component-Based Architecture ที่แบ่งหน้าจอออกเป็นส่วนประกอบย่อย ๆ (Component) ที่สามารถนำกลับมาใช้ซ้ำได้ ทำให้การพัฒนาเว็บแอปพลิเคชันมีความเป็นระบบและบำรุงรักษาง่าย React ใช้ Virtual DOM ในการจัดการการแสดงผล ทำให้สามารถอัปเดตเฉพาะส่วนที่มีการเปลี่ยนแปลงได้อย่างรวดเร็ว โดยไม่ต้องโหลดหน้าเว็บใหม่ทั้งหมด (Single Page Application - SPA) ในปัจจุบัน React เป็นหนึ่งในเครื่องมือพัฒนา Frontend ที่ได้รับความนิยมสูงสุดทั่วโลก (Meta Open Source, 2024, https://react.dev/)

### 2. Vite
&emsp;&emsp;Vite คือ เครื่องมือสร้างโปรเจกต์ (Build Tool) รุ่นใหม่สำหรับการพัฒนาเว็บแอปพลิเคชัน พัฒนาโดย Evan You ผู้สร้าง Vue.js Vite มีจุดเด่นในด้านความเร็วในการเริ่มต้นเซิร์ฟเวอร์สำหรับการพัฒนา (Dev Server) โดยใช้ Native ES Modules ทำให้ไม่ต้องรอการ Bundle ไฟล์ทั้งหมดก่อนเริ่มพัฒนา ส่งผลให้การพัฒนาเป็นไปอย่างรวดเร็วและมีประสิทธิภาพสูง Vite รองรับการทำงานร่วมกับ React, Vue, Svelte และอื่น ๆ ถือเป็นตัวเลือกที่ทันสมัยแทนที่เครื่องมือรุ่นก่อน เช่น Webpack หรือ Create React App (Vite, 2024, https://vitejs.dev/)

### 3. Tailwind CSS
&emsp;&emsp;Tailwind CSS คือ CSS Framework แบบ Utility-First ที่ช่วยให้นักพัฒนาสามารถจัดรูปแบบ (Styling) องค์ประกอบของเว็บไซต์ได้อย่างรวดเร็ว โดยการเขียน CSS Class โดยตรงในไฟล์ HTML หรือ JSX แทนการเขียนไฟล์ CSS แยกต่างหาก Tailwind CSS มีจุดเด่นในด้านความยืดหยุ่นสูง สามารถปรับแต่งธีม (Theme) สี ขนาดตัวอักษร ระยะห่าง และอื่น ๆ ได้ตามต้องการ ช่วยให้การพัฒนาหน้าเว็บมีความรวดเร็วและสม่ำเสมอ รองรับ Responsive Design และ Dark Mode ได้อย่างง่ายดาย (Tailwind Labs, 2024, https://tailwindcss.com/)

### 4. Progressive Web Application (PWA)
&emsp;&emsp;PWA หรือ Progressive Web Application คือ เว็บแอปพลิเคชันที่ถูกพัฒนาให้มีความสามารถใกล้เคียงกับแอปพลิเคชันบนมือถือ (Native App) สามารถติดตั้งลงบนหน้าจอโทรศัพท์มือถือหรือคอมพิวเตอร์ได้โดยไม่ต้องผ่าน App Store หรือ Play Store PWA มีคุณสมบัติสำคัญ ได้แก่ การทำงานแบบออฟไลน์ (Offline Support) การส่งการแจ้งเตือน (Push Notifications) และการโหลดข้อมูลที่รวดเร็ว โดยอาศัย Service Worker และ Web App Manifest ในการจัดการ ในระบบ SEC RMS ได้ใช้ vite-plugin-pwa เพื่อเปิดใช้งานความสามารถ PWA ทำให้ผู้ใช้งานสามารถติดตั้งระบบลงบนมือถือและใช้งานได้สะดวกยิ่งขึ้น (Google Developers, 2024, https://web.dev/progressive-web-apps/)

## ระบบฐานข้อมูลและบริการคลาวด์

### Google Firebase
&emsp;&emsp;Google Firebase คือ แพลตฟอร์มบริการคลาวด์ (Cloud Platform) ที่พัฒนาโดย Google สำหรับการสร้างและบริหารจัดการเว็บแอปพลิเคชันและแอปพลิเคชันบนมือถือ Firebase ให้บริการแบบ Backend-as-a-Service (BaaS) ซึ่งช่วยให้นักพัฒนาไม่จำเป็นต้องสร้างและดูแลเซิร์ฟเวอร์เอง ในระบบ SEC RMS ได้ใช้บริการของ Firebase ดังนี้

#### 1) Cloud Firestore
&emsp;&emsp;Cloud Firestore คือ ฐานข้อมูลแบบ NoSQL ที่จัดเก็บข้อมูลในรูปแบบเอกสาร (Document) และคอลเลกชัน (Collection) รองรับการอ่านเขียนข้อมูลแบบเรียลไทม์ (Real-time) มีความยืดหยุ่นสูงในการจัดเก็บข้อมูลที่มีโครงสร้างซับซ้อน สามารถปรับขนาดรองรับจำนวนผู้ใช้งานที่เพิ่มขึ้นได้อัตโนมัติ (Auto-scaling) ในระบบ SEC RMS ใช้ Firestore เก็บข้อมูลรายการเคลม (RMA), ข้อมูลผู้ใช้งาน, ยี่ห้อสินค้า, ผู้จัดจำหน่าย และการตั้งค่าระบบ

#### 2) Firebase Authentication
&emsp;&emsp;Firebase Authentication คือ บริการยืนยันตัวตนผู้ใช้งานที่รองรับหลายรูปแบบ เช่น การเข้าสู่ระบบด้วยอีเมลและรหัสผ่าน, Google Account, Facebook เป็นต้น มีระบบรักษาความปลอดภัยที่ได้มาตรฐาน ช่วยให้นักพัฒนาสามารถจัดการการเข้าสู่ระบบและสิทธิ์ของผู้ใช้งานได้อย่างสะดวก ในระบบ SEC RMS ใช้ Firebase Authentication สำหรับการเข้าสู่ระบบของเจ้าหน้าที่และผู้ดูแลระบบ

#### 3) Firebase Hosting
&emsp;&emsp;Firebase Hosting คือ บริการโฮสติ้งสำหรับเว็บไซต์แบบ Static และ Single Page Application (SPA) มีความเร็วสูง รองรับ SSL/HTTPS โดยอัตโนมัติ และสามารถ Deploy เว็บไซต์ได้ด้วยคำสั่งเพียงไม่กี่บรรทัด ในระบบ SEC RMS ใช้ Firebase Hosting เป็นแพลตฟอร์มในการเผยแพร่ระบบบนอินเทอร์เน็ต

(Google Firebase, 2024, https://firebase.google.com/)

## โปรแกรมที่ใช้ในการพัฒนาระบบ

### 1. Visual Studio Code (VS Code)
&emsp;&emsp;Visual Studio Code หรือ VS Code เป็นโปรแกรม Code Editor ที่ใช้ในการแก้ไขและปรับแต่งโค้ดจากค่ายไมโครซอฟท์ มีการพัฒนาออกมาในรูปแบบของ Open Source จึงสามารถนำมาใช้งานได้แบบฟรี ๆ ซึ่ง Visual Studio Code นั้นเหมาะสำหรับนักพัฒนาโปรแกรมที่ต้องการใช้งานข้ามแพลตฟอร์ม รองรับการใช้งานทั้งบน Windows, macOS และ Linux สนับสนุนทั้งภาษา JavaScript, TypeScript และ Node.js สามารถเชื่อมต่อกับ Git ได้ นำมาใช้งานได้ง่าย ไม่ซับซ้อน มีเครื่องมือส่วนขยาย (Extension) ต่าง ๆ ให้เลือกใช้อย่างมาก ไม่ว่าจะเป็น 1) การเปิดใช้งานภาษาอื่น ๆ ทั้งภาษา C++, C#, Java, Python, PHP หรือ Go 2) Themes 3) Debugger 4) Commands เป็นต้น (Microsoft, 2024, https://code.visualstudio.com/)

### 2. Node.js และ npm
&emsp;&emsp;Node.js คือ สภาพแวดล้อมการทำงาน (Runtime Environment) ของ JavaScript ที่ทำงานบนฝั่งเซิร์ฟเวอร์ พัฒนาบนเอนจิน V8 ของ Google Chrome ในการพัฒนาระบบ SEC RMS ใช้ Node.js ร่วมกับ npm (Node Package Manager) ซึ่งเป็นเครื่องมือสำหรับจัดการแพ็คเกจ (Package) และไลบรารีต่าง ๆ ที่จำเป็นต่อการพัฒนา เช่น React, Vite, Firebase SDK, Tailwind CSS และอื่น ๆ (Node.js Foundation, 2024, https://nodejs.org/)

### 3. Git และ GitHub
&emsp;&emsp;Git คือ ระบบควบคุมเวอร์ชัน (Version Control System) แบบกระจาย (Distributed) ที่ใช้สำหรับติดตามการเปลี่ยนแปลงของซอร์สโค้ด ช่วยให้ทีมพัฒนาสามารถทำงานร่วมกันได้อย่างมีประสิทธิภาพ สามารถย้อนกลับไปยังเวอร์ชันก่อนหน้าได้หากเกิดข้อผิดพลาด GitHub คือ แพลตฟอร์มที่ให้บริการ Hosting สำหรับ Git Repository บนอินเทอร์เน็ต ช่วยให้สามารถเก็บสำรองโค้ดบนคลาวด์ได้อย่างปลอดภัย (GitHub, 2024, https://github.com/)

## โปรแกรมที่ใช้ในการออกแบบ และจัดทำเอกสาร

### 1. Microsoft Office Word 365
&emsp;&emsp;Microsoft Office Word 365 คือ โปรแกรมประเภท Word Processor ที่เหมาะสำหรับการพิมพ์รายงาน พิมพ์จดหมาย เป็นหนึ่งในโปรแกรมไมโครซอฟท์ออฟฟิศ ซึ่งมีพัฒนาการอย่างต่อเนื่อง ช่วยให้สร้างเอกสารแบบมืออาชีพอย่างมีประสิทธิภาพ ทั้งยังสามารถตรวจสอบ ทบทวน แก้ไข ปรับปรุงความถูกต้องในการพิมพ์เอกสารได้อย่างง่ายดาย สามารถตรวจสอบสะกดคำ และหลักไวยากรณ์ เพิ่มตาราง เพิ่มกราฟิกในเอกสารได้อย่างง่ายดาย

### 2. Microsoft Office Excel 365
&emsp;&emsp;Microsoft Office Excel 365 คือ โปรแกรมประเภท Spreadsheet ที่ใช้สำหรับเก็บข้อมูลในลักษณะของตาราง มีความสามารถในการคำนวณและสร้างกราฟจากข้อมูลได้อย่างรวดเร็ว ง่ายต่อการป้อนข้อมูล การแก้ไขข้อมูล สะดวกต่อการคำนวณและการนำข้อมูลไปประยุกต์ใช้ สามารถจัดข้อมูลต่าง ๆ ได้อย่างเป็นหมวดหมู่และเป็นระเบียบมากยิ่งขึ้น

### 3. Microsoft Office PowerPoint 365
&emsp;&emsp;Microsoft Office PowerPoint 365 เป็นโปรแกรมที่ถูกนำมาใช้เพื่อการนำเสนองานต่าง ๆ ซึ่งมีความยืดหยุ่นสูง สามารถทำสไลด์ในการนำเสนองานได้หลากหลาย มีธีมที่ออกแบบมาให้ หรือจะออกแบบเองก็ได้ สามารถใส่ Animation ต่าง ๆ เพื่อให้เกิดสีสัน และความสวยงามในการนำเสนองานได้เป็นอย่างดี สามารถแทรกรูปภาพต่าง ๆ เข้าไปในการนำเสนอได้อย่างอิสระ มีความสะดวก และง่ายต่อการใช้งาน

### 4. Microsoft Office Project 365
&emsp;&emsp;Microsoft Office Project 365 เป็นโปรแกรมที่ได้ถูกพัฒนาขึ้นมาโดยมีจุดประสงค์เพื่อช่วยในการบริหารโครงการ ซึ่งมีความสามารถในการวางแผน (Schedule) และความสามารถในการคำนวณค่าใช้จ่าย (Cost) รวมถึงการติดตามการทำงานที่ทำสำเร็จไปแล้ว หลักสำคัญของการบริหารโครงการ คือ การวางแผนหลาย ๆ งาน ซึ่งประกอบกันขึ้นมาเป็นโครงการภายในระยะเวลาที่กำหนดไว้
